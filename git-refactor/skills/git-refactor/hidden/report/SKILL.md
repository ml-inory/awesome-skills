# report

Generate `README-fix-report.md` in the project root, summarizing every fix applied during the refactor run. The report file is intentionally left uncommitted.

## Required Inputs

- `fix_log`: The complete list of fix_log_entry objects from all `fix` steps, each now with verification results.
- `project_dir`: Absolute path to the local project.

## Execution

1. Write `README-fix-report.md` to `project_dir/README-fix-report.md`.
2. Structure the report as follows:

```markdown
# README Fix Report

Generated by git-refactor workflow on <date>.

**Branch**: `fix/run-on-ax-board`
**Original Branch**: `<original_branch>`
**Board**: `<chip> @ <hostname> (<ip>)`
**Total Fixes**: <count>

---

## Fix 1: <commit_message>

- **Step**: Step <N> — `<first line of commands>`
- **Category**: <error_category>
- **Root Cause**: <root_cause>
- **Files Changed**:
  - `<file1>`: <what changed>
  - `<file2>`: <what changed>
- **Commit**: `<hash>` — `<commit_message>`
- **Verification**: <passed|partial — <brief note>>

---

## Summary

| # | Step | Category | Root Cause | Verification |
|---|------|----------|------------|--------------|
| 1 | ... | ... | ... | ... |
```

3. After writing, verify the file exists.
4. Ensure the file is NOT staged in git: run `git status -- README-fix-report.md` and confirm it shows as untracked (not in index).

## Outputs

```json
{
  "report_path": "/absolute/path/to/project/README-fix-report.md",
  "total_fixes": 3,
  "not_staged": true
}
```

## Failure Cases

- Cannot write to `project_dir`: fail.
- File is accidentally staged: unstage it with `git rm --cached README-fix-report.md`.

## Hard Constraints

- Do NOT `git add` or `git commit` README-fix-report.md.
- Do NOT push anything remotely.
- If the file already exists from a previous run, overwrite it.
